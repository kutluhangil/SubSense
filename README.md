## рҹҺҜ Why SubSense?

Most people have no idea how much they're actually spending on subscriptions. Trials forgotten. Prices silently raised. Services doubled up without anyone noticing. By the time you check, hundreds of dollars have quietly slipped away each month.

**SubSense fixes this with three simple promises:**

```
 1.  See every subscription in one place
 2.  Know your real total, in your own currency
 3.  Get AI-driven advice on what to cancel
```

## рҹҡҖ Quick Start

### Prerequisites

- Node.js **v18+**
- A Firebase project with **Auth** and **Firestore** enabled
- A Gemini API key вҖ” [get one free here](https://aistudio.google.com/app/apikey)

### Installation

```bash
# Clone
git clone https://github.com/your-username/SubSense.git
cd SubSense

# Install
npm install

# Configure
cp .env.example .env.local
# вҶ’ Fill in your keys (see below)

# Run
npm run dev
```

Open [http://localhost:5173](http://localhost:5173) and you're in.

### Environment Variables

Create a `.env.local` file in the project root:

```env
# Firebase
VITE_FIREBASE_API_KEY=
VITE_FIREBASE_AUTH_DOMAIN=
VITE_FIREBASE_PROJECT_ID=
VITE_FIREBASE_STORAGE_BUCKET=
VITE_FIREBASE_MESSAGING_SENDER_ID=
VITE_FIREBASE_APP_ID=

# AI
VITE_GEMINI_API_KEY=
```

> вҡ пёҸ **Never commit `.env.local`.** It's already in `.gitignore`.

## рҹ“Ұ Deployment

### Option A вҖ” Vercel <sub>(recommended)</sub>

The fastest path to production:

```bash
npm run build   # Verify locally first
```

Connect the repo to Vercel, add your env vars under **Settings вҶ’ Environment Variables**, and deploy. Done.

### Option B вҖ” Self-Hosted <sub>(Docker + Nginx)</sub>

Deploy to your own server using the included Docker setup:

```bash
# On your Ubuntu server
cd SubSense/server

# Build and launch
docker compose up -d --build

# Verify
docker compose ps
docker compose logs nginx --tail=20
```

App serves on port `3002` by default вҖ” configurable in `server/docker-compose.yml`.
